# 单细胞高维向量 ANN 检索系统（详版）

本文保留 README.md 中移出的完整说明，适合需要更深入了解项目背景、协作分工与扩展规划的读者。

## 项目简介

本项目是一个面向单细胞高维向量数据的近似最近邻检索系统。

系统的核心目标是：读取单细胞数据文件，提取细胞向量表示，构建 ANN 近似最近邻索引，并通过 Web 页面提供 Top-K 相似细胞检索功能。在此基础上，集成了 ChromaDB 向量数据库和大模型 RAG（检索增强生成）能力，支持自然语言智能问答。

本项目整体定位为：

> 数据处理与算法为主，前端展示为辅。

## 项目背景

单细胞测序数据通常包含大量细胞样本。经过预处理后，每个细胞都可以表示为一个高维向量，例如基因表达向量或 PCA 降维后的向量。

当细胞数量较大、向量维度较高时，传统精确最近邻搜索的计算成本较高。近似最近邻检索（Approximate Nearest Neighbor, ANN）能够在保证较好检索效果的同时提升查询效率，因此适合用于单细胞相似细胞搜索场景。

本项目通过 FAISS、HNSWLIB 等 ANN 工具，实现对单细胞向量数据的快速 Top-K 相似检索，并已支持 FAISS-PQ。同时引入 ChromaDB 向量数据库和 RAG 技术，支持通过自然语言与数据集进行智能交互。

## 团队分工

本项目采用 **3 个后端 + 1 个前端** 的分工模式，便于并行开发，减少 Git 冲突，并保证每位成员都有清晰的提交记录。

### 后端 1：数据开发 Data Engineer

**职责定位：** 负责单细胞数据读取、解析与向量提取。

**主要技术：** Python、Scanpy、NumPy、Pandas

**负责文件：**

```text
backend/data_reader.py
backend/dataset_preprocessor.py
```

**主要任务：**

1. 使用 `read_h5ad()` 读取 `.h5ad` 文件
2. 提取表达矩阵 `X`、PCA 向量 `obsm/X_pca`、细胞元数据 `obs`
3. 将向量转换为标准 `float32` NumPy 数组
4. 封装 `DataLoader` 类（内存高效懒加载）
5. 封装 `DatasetManager` 多数据集管理类
6. 实现多数据集预处理管线（清洗、QC 过滤、基因对齐、合并）

---

### 后端 2：算法开发 Algorithm Engineer

**职责定位：** 负责 ANN 索引构建和相似细胞检索逻辑。

**主要技术：** Python、FAISS / HNSWLIB、NumPy

**负责文件：**

```text
backend/ann_indexer.py
```

**主要任务：**

1. 接收数据模块输出的细胞向量矩阵
2. 构建 ANN 检索索引（支持 FAISS/HNSWLIB/NumPy 三种后端）
3. 实现 Top-K 相似细胞检索
4. 返回相似细胞编号和距离
5. 支持索引保存到硬盘和加载
6. 支持多种索引类型和距离度量

---

### 后端 3：Web 接口开发 API Developer

| 角色 | 权限说明 |
|---|---|
| 普通用户 `user` | 注册登录、查看可视化系统、执行相似检索、查看数据集、使用 AI 助手、查看性能评测、维护个人资料和密码 |

### 安全策略

- 密码不会明文保存，统一使用 Werkzeug `pbkdf2:sha256` 哈希函数写入数据库
- 页面访问通过 Flask Session 识别当前用户（有效期 7 天）
- `/users` 页面和 `/api/users` 接口仅管理员可访问
- 数据上传、数据删除、索引构建和索引删除接口仅管理员可调用
- 向量库初始化和清空仅管理员可调用
- 系统阻止删除、禁用或降级最后一个启用的管理员，避免管理员锁死

## 核心模块说明

### data_reader.py

负责读取和解析单细胞数据。

- `DataLoader`：加载单个 `.h5ad` 文件，提供向量矩阵和细胞元数据读取接口
- `DatasetManager`：多数据集管理器，支持增删查、上传校验和元信息维护

### ann_indexer.py

负责 ANN 索引构建和检索。

- `ANNIndexer`：ANN 索引构建、Top-K 检索、索引保存与加载
- `IndexConfig`：索引配置数据类，支持从环境变量和 API 参数构建

### vector_store.py

负责 ChromaDB 向量数据库封装。

- `CellVectorStore`：将 DataLoader 中的细胞向量批量写入 ChromaDB，提供向量相似检索、元数据过滤和关键词检索接口

### llm_client.py

负责大模型 API 接入。

- `LLMClient`：统一封装智谱 GLM SDK 和 OpenAI SDK，提供 `chat()`（阻塞）、`stream_chat()`（流式）和 `embed()` 接口

### rag_engine.py

负责 RAG 完整流程编排。

- `RAGEngine`：串联向量检索、Prompt 组装和 LLM 生成，管理多轮对话历史
- `ChatHistory`：内存对话历史管理，支持 TTL 过期清理

### prompt_builder.py

负责 Prompt 工程。

- `PromptBuilder`：将检索到的细胞数据（类型/基因/元数据）格式化为结构化文本上下文
- `PromptTemplate`：可配置的 Prompt 模板数据类
- 内置三种角色预设：生信分析专家、严谨数据统计员、通俗科普助手

### dataset_preprocessor.py

负责多数据集预处理管线。

- 支持加载、清洗、QC 过滤、标准化、log1p 变换和基因对齐合并
- 输出合并后的 AnnData 文件和 JSON 处理报告

### app.py

负责启动后端 Web 服务。

- 初始化 Flask 应用和用户数据库
- 管理登录会话和角色权限
- 提供 40+ 个 REST API 路由
- 连接数据读取、ANN 检索、ChromaDB 向量库和 RAG 引擎

### user_store.py

负责用户数据持久化和认证逻辑。

- SQLite 存储：用户表、搜索快照表、对话会话表、对话消息表
- 密码哈希存储、登录验证、角色权限控制
- 搜索快照持久化、对话消息持久化

```python
vectors.shape == (cell_count, vector_dim)
vectors.dtype == np.float32
```

## 项目亮点

1. **模块分工清晰**

   数据读取、算法检索、Web API、前端展示分别由不同成员负责，代码结构清楚，便于协作开发。

2. **核心算法突出**

   项目重点实现单细胞高维向量的 ANN 检索，支持 FAISS/HNSWLIB/NumPy 三种后端和五种索引类型，体现数据处理和算法能力。

3. **RAG AI 智能问答**

   集成 ChromaDB 向量数据库和大模型 RAG 技术，支持自然语言与数据集智能交互，具备流式回答、多轮对话和溯源展示能力。

4. **多算法性能评测**

   自动对比五种 ANN 算法的延迟和召回率，以可视化图表直观展示各算法在不同 K 值下的性能差异。

5. **UMAP 交互可视化**

   基于 ECharts 的交互式 UMAP 散点图，支持缩放、分类着色、查询细胞高亮和检索结果标注。

6. **搜索快照与重跑**

   每次搜索自动保存参数快照，用户可在个人中心查看历史搜索并一键重跑，便于实验复现。

7. **工程实践完整**

   项目包含数据模块、算法模块、接口模块、前端模块、向量数据库和 RAG 引擎，形成完整的软件系统闭环。

8. **便于并行开发**
